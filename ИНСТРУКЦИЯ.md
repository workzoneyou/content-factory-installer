# Установка — пошагово (что делать и как отвечать)

Полное руководство: от чистого сервера до работающего блога. Установщик задаёт
**всего 5 вопросов** — всё остальное (пароли, ключи, тема, плагины) делается само.

---

## ⚠️ Главное: где это запускается

Установщик работает **на сервере (Ubuntu), а НЕ на вашем компьютере/Маке.**
Блог и n8n живут на арендованном сервере — это компьютер в дата-центре, работает
круглосуточно. Ваш Мак нужен только чтобы **подключиться** к серверу и запустить установку.

Порядок: **арендовать сервер → направить домены → подключиться к серверу → запустить установщик на сервере.**

---

## Шаг 0. Арендовать сервер (VPS)

Любой хостинг VPS (Timeweb, Aeza, Beget, reg.ru и т.п.). Выбрать:
- **Ubuntu 22.04** (или новее),
- 2 ГБ RAM (лучше 4), ~40 ГБ SSD.

После оплаты на почту придут **IP-адрес сервера** и **пароль root**.
*(Если сервер уже есть — переходите к шагу 1.)*

---

## Шаг 1. Направить домены на сервер (DNS)

В панели домена создать **две A-записи**, обе на **IP вашего сервера**:

| Тип | Имя (хост) | Значение |
|-----|-----------|----------|
| A | `blog` (→ `blog.вашдомен.ru`) | IP сервера |
| A | `n8n`  (→ `n8n.вашдомен.ru`)  | IP сервера |

⚠️ Сделать это **до** установки. Без A-записей Let's Encrypt не выпустит SSL —
сайт откроется с ошибкой сертификата, пока записи не «долетят» (обычно 5–30 минут).

---

## Шаг 2. Подключиться к серверу с вашего компьютера

Открыть **Терминал** (на Маке — программа «Терминал»; на Windows — PowerShell) и ввести:

```bash
ssh root@IP_сервера
```

- Первый раз спросит подтверждение — напечатать `yes`, Enter.
- Спросит пароль root (из письма хостинга). **При вводе пароль не виден — это нормально**,
  просто наберите и нажмите Enter.
- Приглашение сменится на `root@...` — значит вы **на сервере**. Дальше команды выполняются тут.

---

## Шаг 3. Запустить установщик (уже на сервере!)

Вы под root, поэтому **без `sudo`**:

```bash
curl -sSL https://raw.githubusercontent.com/workzoneyou/content-factory-installer/main/setup.sh | bash
```

> Эту команду набирать **в окне, где вы подключены к серверу** (приглашение `root@...`),
> а НЕ в обычном окне своего компьютера. На Маке/Windows напрямую она не работает.

Дальше установщик задаёт вопросы прямо в терминале. Отвечаете — жмёте **Enter**.
Где в конце вопроса показано `[значение в скобках]` — это значение по умолчанию:
можно просто нажать Enter, оно подставится.

---

## Шаг 1. Пять вопросов установщика

### Вопрос 1 — Домен блога
```
? Домен блога (напр. blog.site.ru):
```
Адрес, по которому откроется сам блог. Вводить **без `https://`** и без `/` в конце —
установщик сам почистит. Обязательный, пустым оставить нельзя.
- **Пример ответа:** `blog.ivanov.ru`

### Вопрос 2 — Домен n8n
```
? Домен n8n (напр. n8n.site.ru):
```
Адрес панели n8n (движок, который генерирует статьи). Тоже без `https://`.
- **Пример ответа:** `n8n.ivanov.ru`

### Вопрос 3 — Email для SSL
```
? Email для SSL-сертификата (Let's Encrypt):
```
Реальная почта. На неё привязывается аккаунт бесплатного сертификата Let's Encrypt
(туда изредка падают уведомления, если продление сломается). Проверяется формат.
- **Пример ответа:** `me@ivanov.ru`

### Вопрос 4 — Логин WP-админа
```
? Логин WP-админа [admin]:
```
Логин для входа в админку WordPress. Нажать Enter → будет `admin`.
- **Рекомендация:** для безопасности лучше **не** `admin`, а что-то своё.
- **Пример ответа:** `ivanov_adm` (или просто Enter для `admin`)

### Вопрос 5 — Email WP-админа
```
? Email WP-админа [me@ivanov.ru]:
```
Почта администратора WordPress. К ней же привязывается **аватар (граватар)**.
Нажать Enter → подставится почта из вопроса 3.
- **Рекомендация:** укажите ту почту, на которую потом заведёте граватар для аватарки автора.
- **Пример ответа:** Enter (или отдельная почта)

**Всё. Больше вопросов нет.** Пароли WP-админа, базы данных и ключ шифрования n8n
установщик генерирует сам.

---

## Шаг 2. Что происходит дальше само (ничего вводить не нужно)

1. Проверяет DNS, при необходимости ставит Docker и firewall (порты 22/80/443).
2. Скачивает компоненты в папку `~/firsanov-factory`.
3. Поднимает WordPress: ставит сам WP, русский язык, ЧПУ-ссылки `/%postname%/`,
   активирует тему **firsanov**, ставит плагины (Yoast, кэш, IndexNow, Метрика, GraphQL).
4. Применяет фиксы: пропорцию картинок 3/2, права на загрузки, соцполя профиля,
   Yoast noindex автор-архивов.
5. Поднимает n8n (пустой).

Занимает ~2–5 минут. В конце появится табличка:

```
✓ Готово
  Блог:    https://blog.ivanov.ru   (ivanov_adm / <сгенерированный пароль>)
  n8n:     https://n8n.ivanov.ru
  Секреты — в /root/firsanov-factory/.env  (сохраните в бэкап!)
```

⚠️ **Скопируйте пароль админа и сохраните.** Он и все остальные секреты лежат в файле
`~/firsanov-factory/.env` — этот файл тоже стоит забэкапить.

Первый SSL выпускается 30–60 секунд. Проверить: `curl -sI https://blog.ivanov.ru`
(должен ответить `HTTP/2 200`). Если ошибка сертификата — подождать пару минут (DNS/ACME).

---

## Шаг 3. Настроить в самом WordPress (`https://blog.вашдомен.ru/wp-admin`)

Эти вещи специально **не** спрашивались при установке — их удобнее задать в админке.

1. **Название сайта** → Настройки → Общие → «Название сайта».
2. **Профиль автора** → Пользователи → ваш профиль:
   - Отображаемое имя, «Биографическая информация» (описание автора).
   - Поля **ВКонтакте / Telegram / RuTube / TenChat** — уже добавлены, просто вписать ссылки.
   - Аватар подтянется автоматически, если завести **граватар** на почту админа
     (шаг 5 установки) на gravatar.com.
3. **Яндекс.Метрика** → в настройках плагина метрики вставить ID счётчика.

---

## Шаг 4. Настроить в n8n (`https://n8n.вашдомен.ru`)

Движок ставится пустым — воркфлоу и ключи заводятся руками.

1. **Импорт воркфлоу**: n8n → Import from File (загрузить json-файлы пайплайна).
2. **Credentials** — завести и выбрать в нодах с красным значком:
   - Google Sheets (OAuth2), OpenRouter, KIE (Header Auth `Authorization=Bearer <key>`),
     Perplexity, WordPress (Basic Auth: логин + пароль приложения).
3. **Google-таблица**: подключить лист. ⚠️ Лист **ОПУБЛИКОВАНО оставить пустым**
   (иначе внутренние ссылки в статьях будут вести на несуществующие URL → 404).
4. **Контент**: залить темы/офферы → сделать тест-прогон МАСТЕРА.

---

## Если что-то пошло не так

| Симптом | Причина / что делать |
|---------|----------------------|
| Ошибка сертификата в браузере | DNS ещё не долетел или SSL выпускается. Подождать 2–5 мин, обновить. |
| `Домен … пока не направлен на сервер` при установке | Проверьте A-записи (шаг 0). Установка продолжится, но SSL выпустится только после DNS. |
| Забыли пароль админа | Он в файле `~/firsanov-factory/.env`, строка `WP_ADMIN_PASSWORD`. |
| Логи | `cd ~/firsanov-factory/wordpress && docker compose logs -f wp` (или `../n8n` для n8n). |
